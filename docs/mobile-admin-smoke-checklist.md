# Smoke-чеклист мобильной админ-панели

## Навигация
- [ ] `MobileAppBar`: отображается на ширине < 768px, кнопка меню открывает `MobileAdminDrawer`
- [ ] `MobileAdminDrawer`: содержит актуальные вкладки, бейджи чата/заявок обновляются при непрочитанных событиях
- [ ] FAB: контекстно меняется при переключении вкладок (`overview`, `users`, `schools`, `services`)
- [ ] Desktop header: при ширине ≥ 768px отображается стационарный заголовок и табы, без дубля MobileAppBar

## Обзор (Dashboard Overview)
- [ ] Collapsible блок статистики: на мобильных по умолчанию показаны top-3 карточки, кнопка «Показать все» раскрывает остальные
- [ ] На десктопе блок статистики раскрыт, сетка отображает 1/2/3/4/5 колонок в зависимости от брейкпоинта
- [ ] Carousel «Быстрые ссылки»: горизонтальный скролл, карточки ведут на соответствующие вкладки
- [ ] «Быстрые действия»: кнопки создают пользователя, школу, услугу и переключают на нужную вкладку

## Пользователи
- [ ] Mobile: `ResponsiveList` + `UserCard`, фильтры открываются в `FilterDrawer`, selection работает с `BulkActionBar`
- [ ] Desktop: таблица остаётся прежней, поиск/кнопки не ломают layout

## Школы
- [ ] Mobile: отображение карточек `SchoolCard`, действия редактирования/удаления открывают соответствующие модалы
- [ ] Desktop: таблица и фильтры работают как ранее

## Мастер-классы
- [ ] Mobile: карточки `MasterClassCard`, раскрытие деталей ведёт в bottom sheet
- [ ] `MasterClassDetails`: вкладки (Обзор/Участники/Оплаты) работают в мобильном bottom sheet и десктопном диалоге

## Счета
- [ ] Mobile: карточки `InvoiceCard`, bottom sheet деталей счёта открывается корректно
- [ ] Проверка оплат через Robokassa (кнопка оплаты → корректная ссылка/форма)

## Заявки
- [ ] Mobile: карточки `RequestCard`, статусы меняются, counters в Dashboard обновляются (pending/approved/rejected)
- [ ] WebSocket обновляет статистику без перезагрузки страницы (индикатор подключения зелёный)

## Чат
- [ ] Mobile: список диалогов `ChatDialogCard`, unread badge обновляется; переход в чат показывает сообщения
- [ ] Desktop: двухколоночный layout, новые сообщения подсвечивают таб «Чат»

## Робокасса / Платежи
- [ ] Открытие оплаты на странице завершения записи → Robokassa (проверка fallback на HTML-форму)
- [ ] Детали заказа → кнопка оплаты открывает Robokassa, статус счёта обновляется после успешной оплаты

## Общее
- [ ] Авторизация/логаут администратора работают как до изменений
- [ ] Проверка Lighthouse на mobile: FCP < 1.5s, LCP < 2.5s
- [ ] Нет ошибок в консоли браузера/Network 4xx/5xx на основных сценариях
